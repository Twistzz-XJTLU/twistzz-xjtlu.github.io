Week1

Data Types

Five different data types

  • item: an object
  • link: relationship between items
  • attribute: property of an item
  • position: a location in 2D or 3D space
  • grid: regular sampling of continuous data

Four different data set types

  • table: rows and columns (2D or multidimensional)
  • networks and trees: relationships between items
  • fields: continuous data (conceptually there are an infinite number of measurements you could take, so sampling and extrapolation is necessary)
  • geometry: spatial data

Data Availability

  • static
  • dynamic

Attributes

  • categorical
  • ordered(判断的方法 看是不是数字的 可不可以做加减法)
    • ordinal
    • quantitative

  • diverging

Week3

The Semiology of Graphics

Bertin’s Visual Variables

  • Location variables (position, relative to a coordinate frame)
    • e.g. horizonal and vertical axes on a scatterplot; longitude and latitude on a map
    • (so fundamental to presenting map information that these variables are often ignored in cartography)
  • Retinal variables (perceptual properties)
    • ways of representing differences between objects
    • size, shape, colour (hue), colour (value), texture, orientation

The Six Retinal Variables

  • Shape: (e.g. square, circle, star)
  • Size: (e.g. measured in mm or pixels)
  • Orientation: angle of most prominent axis in the symbol to the coordinate axes
  • Texture: spacing between repeated elements of a symbol (e.g. fine, coarse)
  • Hue: colour, as associated with wavelength (e.g. blue, green, turquoise)
  • Value: depth of colour, as associated ink density and represented by greyscale (e.g. red ink with low value will be perceived as pink)

Pre-attentive processing

  • Associative variables

    All variations are perceived equally (location, shape, orientation, colour hue, texture)

  • Dissociative variables

    One variation dominates (size, colour value)

  • Selective variables

    only shape is not selective

  • Ordered perception

    Variations are perceived as being ranked in order (location, size, colour value)

  • Quantitative perception

    (an extension of ordered perception) The variation can be quantitatively estimated (location, size)

Using the variables

  • Unordered (colour hue, orientation, shape, texture) for nominal information: apples, oranges, pears
  • Ordered, non-quantitative (colour value) for ordinal information: rainfall map
  • Ordered, quantitative (location, size) for numerical information: electricity usage (also good for non-quantitative and nominal information given their visual dominance)

Visual Perception

Three levels to perceiving a scene:

  • Level 1: processing low-level properties (parallel)
  • Level 2: pattern recognition (sequential)
  • Level 3: target-oriented search (sequential)

Level 1: Colour (bottom-up)

  • pre-attention
  • colour

Level 2: Pattern identification (bottom-up & top-down)

  • pattern identification

  • Level 2: Gestalt laws

    • proximity

    • similarity

    • connectedness

    • continuity

    • symmetry

    • closure

    • figure and ground

    • common fate

Level 3: Object identification (top-down)

Week4

Information visualisation pipeline

Types of interaction

  • Filtering: only show me the data I am interested in
  • Selecting: mark or track items I am interested in
  • Abstract & Elaborate: show me more or less detail
  • Overview & Explore/Focus & Context: overview first, zoom and filter, details on demand
  • Connect/Relate: show me how this data is related
  • Reconfigure: show me a different arrangement of the data
  • Encode: show me a different representation of the data
  • Extraction of features: allow me to extract data that interests me
  • History: allow me to retrace the steps I take
  • Participation/Collaboration: allow me to contribute to the data
  • Gamification: show me the data in a more playful way

Data Selection

Basic Pointing Methods

  • Point Selection

    • Mouse Hover / Click

    • Touch / Tap

    • Select Nearby Element (e.g., Bubble Cursor)

  • Region Selection

    • Rubber-band or Lasso
    • Area Cursors(“Brushes”)

Selection by query

  • Dynamic queries: using sliders, checkboxes, etc. to define the query, so it’s easy to explore the data visually
  • Generalised selection: using semantic structure of the data, to explore data at different levels of abstraction
