回溯篇

回溯法解决的问题

  • 组合问题：N 个数里面按一定规则找出 k 个数的集合
  • 切割问题：一个字符串按一定规则有几种切割方式
  • 子集问题：一个 N 个数的集合里有多少符合条件的子集
  • 排列问题：N 个数按一定规则全排列，有几种排列方式
  • 棋盘问题：N 皇后，解数独等等

回溯法解决的问题都可以抽象为树形结构

因为回溯法解决的都是在集合中递归查找子集，集合的大小就构成了树的宽度，递归的深度，都构成的树的深度。

回溯就是在 for 循环中进行递归调用当前函数 for+递归

回溯法模板

def backtracking(参数) {
    if (终止条件):
        存放结果
        return


    for (选择：本层集合中元素（树中节点孩子的数量就是集合的大小）):
        处理节点
		backtracking(路径，选择列表) // 递归
        回溯，撤销处理结果
}

77. 组合

思路

  • startIndex 是本题的关键 这个参数用来记录本层递归的中，集合从哪里开始遍历 从 1 开始
  • 为什么要有这个 startIndex 呢？startIndex 就是防止出现重复的组合
  • 数组是[1,2,3,4] 假设我们当取了 1 那么下一层的遍历就不能取 1 了 因为本题的要求就是取重复的
  • 横向可以理解为可以选择的集合 也就是 for 循环 纵向可以理解为递归进行的深度
  • 函数终止条件就是到达叶子节点 也就是长度是 k 的时候 就是收集结果的时候
  • 为什么要进行回溯撤销节点
  • 因为假设我们最上层是 1 2 3 当前我们在 for 循环中首先取了 1 然后以 1 位根节点进行递归 会进行未知次数的递归
  • 当我们以 1 位根节点进行递归调用完成后 我们还要以 2 为根节点进行递归 但是由于刚才我们在 path 中添加了 1 进去
  • 所以我们要把刚才的 1pop 出来 也就是当我们以当前的根节点的树完全遍历完之后就要移除当前的根节点 进行 pop 清除

代码

class Solution:
    def combine(self, n: int, k: int) -> List[List[int]]:
        result = []  # 存放结果集
        self.backtracking(n, k, 1, [], result)
        return result
    def backtracking(self, n, k, startIndex, path, result):
        if len(path) == k:
            result.append(path[:])
            return
        for i in range(startIndex, n + 1):  # 需要优化的地方
            path.append(i)  # 处理节点
            self.backtracking(n, k, i + 1, path, result)
            path.pop()  # 回溯，撤销处理的节点

17. 电话号码的字母组合

思路

  • 本题的难点 for 循环是循环的什么 整体的递归是如何进行的
  • 首先的问题 假如输入的是 “23” 我们如何遍历输入中的元素的 换句话我们如何遍历 23 呢
  • 为了解决这个问题 我们需要一个指针指向我们当前遍历的元素的位置
  • 也就是now_index这个代表着我们当前遍历的输入 str 的哪个位置了
  • 由此就引出了递归终止条件 也就是如果当前指针的值是输入字符串的长度 那就意味着我们已经遍历完所有元素了
  • 到了叶子节点 要把一条 path 中的值加入的结果集了
  • 那为什么是等于长度呢 假设我们输入 23 当指针是 1 的时候此时我们还没有对指针为 1 的元素进行收集 我们是先判断后进行收集
  • 因此当指针是 2 的时候 才意味着我们已经对指针为 0 和指针为 1 的元素进行了收集
  • 下一个问题: for 循环循环的是谁呢? 我们知道 for 循环是横向遍历的
  • 在本题中我们横向就是当前索引节点的元素(比如索引 0 当前节点是 1) 对应的字母表 也就是 abc
  • 我们要横向遍历 abc 三个字母 分别以 abc 三个字母为根节点继续遍历 继续遍历什么呢
  • 继续遍历下一个索引对应的元素 获取下一个索引对应的元素对应的字母表们
  • 注意本题和前两题不一样 前两天我们用的start_index记录的是横向遍历的开始节点 因为我们的横向遍历的元素是越用越少的
  • 但是本题不一样 我们每个横向遍历的节点都是不一样的 上一层是上一层索引元素对应的字母们
  • 当前层是当前层索引元素对应的字母们
  • 这一题和前两题的根本区别就是我们要遍历并收集的元素集合并不是一个集合
  • 本题如果都不是一个集合了我们当然就要从每个节点的集合中第一个元素开始进行取值然后继续遍历
  • 最后总结一下 我们整体的递归是对输入字符串中的每个元素进行递归 for 循环循环的是当前索引对应的字符对应的字母集
  • 此外别忘了特殊情况(什么也有)需要单独判断

代码

class Solution:
    def __init__(self):
        self.look_up=["","","abc","def","ghi","jkl","mno","pqrs","tuv","wxyz"]
        self.res=[]
        self.path_str=""
    def letterCombinations(self, digits: str) -> List[str]:
        if not digits:
            return []
        self.backtracking(digits,0)
        return self.res
    def backtracking(self,digits,now_index):
        if now_index==len(digits):
            self.res.append(self.path_str)
            return
        digit_index=int(digits[now_index])
        strs=self.look_up[digit_index]
        for i in range(0,len(strs)):
            self.path_str+=strs[i]
            self.backtracking(digits,now_index+1)
            self.path_str=self.path_str[:-1]
        return

39. 组合总和

思路

  • 注意本题是可以使用重复元素的 candidates 中的数字可以无限制重复被选取 这是本题的关键
  • 那这意味着什么呢 意味着我们往下进行竖向遍历的时候 比如 待选集是 123 当我们选了 1 之后下一层还可以选 123
  • 那不可重复使用元素意味着什么呢 意味着当我们选了 1 之后 我们下一层之能选 23 了就不能选 1 了
  • 那可重复和不可重复如何用代码体现呢
  • 体现在递归调用backtracking函数时 startIndex加不加 1
  • 如果是不可重复startIndex也就是i 因为当我们在 for 循环当中startIndex是根据i进行取值的
  • 不可重复是i+1 可重复就是i
  • 那我们横向呢 注意我们最终结果中不能有重复的元素(组合情况下) 比如 12 和 21 是重复的只能算一个
  • 那我们去重呢 这是我们就需要横向遍历来进行去重了 也就是在 for 循环当中
  • 假设待选集是 123 从层的角度去看 当我们第一层首先选取了 1 遍历完 1 位根节点的之后就不能再选 1 了 因为我们要去重
  • 那在代码中如何体现呢 就是for i in range(startIndex, len(candidates))
  • 就是我们要从startIndex开始 意味着我们每次可以选以上一层节点作为开始的元素(分两种情况元素可不可以重复选择)
  • 不能从 0 开始 从零开始意味着我们每次都可以选 123

代码

class Solution:

    def backtracking(self, candidates, target, total, startIndex, path, result):
        if total > target:
            return
        if total == target:
            result.append(path[:])
            return

        for i in range(startIndex, len(candidates)):
            total += candidates[i]
            path.append(candidates[i])
            self.backtracking(candidates, target, total, i, path, result)  # 不用i+1了，表示可以重复读取当前的数
            total -= candidates[i]
            path.pop()

    def combinationSum(self, candidates, target):
        result = []
        self.backtracking(candidates, target, 0, 0, [], result)
        return result

40. 组合总和 II

class Solution:
    def combinationSum2(self, candidates: List[int], target: int) -> List[List[int]]:
        self.res=[]
        self.path=[]
        candidates.sort()
        self.backtracking(candidates,target,0)
        return self.res

    def backtracking(self,candidates,target,start_index):
        if sum(self.path)==target:
            self.res.append(self.path[:])
            return
        if sum(self.path)>target:
            return

        for i in range(start_index,len(candidates)):
            if i>start_index and candidates[i]==candidates[i-1]:
                continue
            self.path.append(candidates[i])
            self.backtracking(candidates,target,i+1)
            self.path.pop()

131. 分割回文串

思路

  • 本体的关键是把 index 当成一把刀进行切割
  • index 的就代表着刀切割的位置 可以理解为 index 是几 就把当前元素和左边的都进行切割 包括当前元素
  • 123 比如 index 是 0 那么就是 1 23 index 是 1 那么就是 12 3 index 就是切割的位置
  • 也可以理解为 index 位置和一下一个元素之间进行切割 比如在 index 是 0 的时候 12 之间进行切割
  • 还有一个关键的就是s[start_index: i + 1] == s[start_index: i + 1][::-1]
  • 判断是否是回文子串 一定要理解为什么是start_index: i + 1 这个带入具体的值比较好理解
  • start_index是什么 是开始的位置 这个一定是我们的父节点传给我们的(一开始是 0) 由于这个是父节点传给我们的
  • 因此这个就是开始的节点假设 123 3 个元素 我们首先切割 1 也就是 index 是 0
  • 当我们获取到 1 的所有情况之后 我们就要获取 2 的所有情况
  • 此时我们 index 是 1 我们要判断回文子串就是 12 了 而不是 1 因为是 2 左边所有的
  • 层层递归 假设 1214 1 234 进入递归 此时start_index就是 1 了 我们的最左边界就变成了start_index

代码

class Solution:

    def partition(self, s: str) -> List[List[str]]:
        result = []
        self.backtracking(s, 0, [], result)
        return result

    def backtracking(self, s, start_index, path, result ):
        # Base Case
        if start_index == len(s):
            result.append(path[:])
            return

        # 单层递归逻辑
        for i in range(start_index, len(s)):
            # 若反序和正序相同，意味着这是回文串
            if s[start_index: i + 1] == s[start_index: i + 1][::-1]:
                path.append(s[start_index:i+1])
                self.backtracking(s, i+1, path, result)   # 递归纵向遍历：从下一处进行切割，判断其余是否仍为回文串
                path.pop()             # 回溯

78. 子集

思路

  • 自己问题和之前的组合问题不一样 有哪里不一样
  • 不一样在于组合问题我们要收集的叶子节点 而子集问题我们要收集的所有节点
  • 首先由于还是不能重复我们需要start_index 让i每次从start_index开始
  • 其次,递归调用i的时候i需不需要加 1 这是需要的 为什么 因为我们元素是不可重复选的 比如 123 我们选了 1 就只能选 2 了
  • 然后就是本题和其他题不一样的地方了 像之前的题我们收集元素的时候都是需要判断是不是叶子节点 有一个条件 满足了该条件我们才会执行res.append(path[:])这样的代码
  • 但是本题是子集问题我们要收集路径上的每一个节点 因为我们就不需要进行判断一路收集节点即可
  • 比如集合收集 123 我们选选了 1 然后 1 加入 path 然后 1 加入 res 然后 2 加入 path 注意此时 path 是 12 而不是只有 2 然后 12 加入 res
  • 然后 3 加入 path 然后 123 加入 res 然后就是向上回溯了因为此时已经遍历到了叶子节点
  • 首选 pop3 然后 pop2 然后加入 3 以此类推

代码

class Solution:
    def subsets(self, nums):
        result = []
        path = []
        self.backtracking(nums, 0, path, result)
        return result

    def backtracking(self, nums, startIndex, path, result):
        result.append(path[:])  # 收集子集，要放在终止添加的上面，否则会漏掉自己
        # if startIndex >= len(nums):  # 终止条件可以不加
        #     return
        for i in range(startIndex, len(nums)):
            path.append(nums[i])
            self.backtracking(nums, i + 1, path, result)
            path.pop()

90. 子集 II

思路

  • 本题和上一题最大的区别是去重
  • 那么我们如何去重呢? 关键的语句是i>start_index and nums[i]==nums[i-1]
  • 这句话代表我们只取最左边那一条路径 在左右两个根节点相同的情况下

代码

class Solution:
    def subsetsWithDup(self, nums: List[int]) -> List[List[int]]:
        nums.sort()
        res,path=[],[]
        self.backtracking(nums,res,path,0)
        return res

    def backtracking(self,nums,res,path,start_index):
        if start_index>len(nums):
            return
        res.append(path[:])

        for i in range(start_index,len(nums)):
            if i>start_index and nums[i]==nums[i-1]:
                continue
            path.append(nums[i])
            self.backtracking(nums,res,path,i+1)
            path.pop()

93. 复原 IP 地址

思路

  • 本题类似分隔回文串
  • +"."注意加.的位置 一定是要在str_path+sub_str后面
  • 最后结果集合元素是不可重复的选择所以for i in range(start_index,len(s))
  • 待选集合元素不可重复 因此递归调用的时候需要i+1
  • for 循环起始的元素控制最后结果集可不可以重复 比如 12 和 21 是重复的 判断完 1 位根节点的元素元素后 以 2 位根节点进行判断的时候就不能再选 1 了 以 2 为起始元素
  • 递归调用i+1j 还是i如果是带选集和可以重复选取那么就是i如果带选集和不可以重复选取那么就是i+1 比如 123 以 1 为根节点进行往下搜索 选了 1 就之能选 23 了 如果可以重复就是选了 1 还是可以选 123

代码

class Solution:
    def restoreIpAddresses(self, s: str) -> List[str]:
        res=[]
        cur_sub_ip=""
        cur_point_num=0
        start_index=0
        self.backtracking(s,res,cur_sub_ip,cur_point_num,start_index)
        return res

    def backtracking(self,s,res,cur_sub_ip,cur_point_num,start_index):
        if cur_point_num==3:
            if self.is_valid(s,start_index,len(s)-1):
                cur_sub_ip+=s[start_index:]
                res.append(cur_sub_ip)
            return
        for i in range(start_index,len(s)):
            if self.is_valid(s,start_index,i):
                sub_str=s[start_index:i+1]
                self.backtracking(s,res,cur_sub_ip+sub_str+".",cur_point_num+1,i+1)
            else:
                break

    def is_valid(self,s,left,right):
        if left>right:
            return False
        if s[left]=="0" and left!=right:
            return False
        num=int(s[left:right+1])
        return num>=0 and num<=255

class Solution:
    def findSubsequences(self, nums):
        result = []
        path = []
        self.backtracking(nums, 0, path, result)
        return result

    def backtracking(self, nums, startIndex, path, result):
        if len(path) > 1:
            result.append(path[:])  # 注意要使用切片将当前路径的副本加入结果集
            # 注意这里不要加return，要取树上的节点

        uset = set()  # 使用集合对本层元素进行去重
        for i in range(startIndex, len(nums)):
            if (path and nums[i] < path[-1]) or nums[i] in uset:
                continue

            uset.add(nums[i])  # 记录这个元素在本层用过了，本层后面不能再用了
            path.append(nums[i])
            self.backtracking(nums, i + 1, path, result)
            path.pop()

46. 全排列

思路

  • 本题是排列题 和之前做的组合题目不一样了 在哪理不一样了 比如 12 和 21 是两种答案 组合的话 12 和 21 就是一种答案
  • 组合的时候我们的 for 循环每次都是从start_index开始 组合就不一样了 我们不能这样 因为第一次我们选了 1 当以 2 为根节点的时候还可以选 1 因此这就是不同之处
  • 但排列问题需要一个 used 数组，标记已经选择的元素 因为我们每个元素还是之能选一次 比如选了 1 以 1 位根节点之后之能选 23 了 1 就不能选了
  • 此时我们的 for 循环开始就是每次都是以 0 开始了也就是第一个元素
  • 进入 for 循环后我们首先要判断当前元素有没有被使用过 如果没有我们就可以选取 有的话就跳过
  • 为什么 for 循环每次要从 0 开始呢 举个例子 下图中 2 位根节点开始的子树 一开始我们选了 2 但是一共有 3 个元素 123 我们还可以选 1 和 3
  • 选 1 意味着什么意味着我们每次都要从带选集中的第一个元素开始寻找看看它可不可以
  • 这也是排列和组合区别的地方 组合的话 当我们选 2 之后 我们只能选 2 之后的了 因为如果我们此时又选了 1 比如 21 但是我们在 1 为根节点的树中已经有了 12 12 和 21 是重复的 因此组合问题要start_index 排列问题不需要每次都从 0 开始
  • used标记数组的初始化要都初始化为 False 因为一开始都没有使用过
  • 最后 used 也要回溯 就是 1 位根节点的树遍历完之后要把 1 元素设置为未使用 因为此时我们要遍历 2 位根节点的元素了

代码

class Solution:
    def permute(self, nums):
        result = []
        self.backtracking(nums, [], [False] * len(nums), result)
        return result

    def backtracking(self, nums, path, used, result):
        if len(path) == len(nums):
            result.append(path[:])
            return
        for i in range(len(nums)):
            if used[i]:
                continue
            used[i] = True
            path.append(nums[i])
            self.backtracking(nums, path, used, result)
            path.pop()
            used[i] = False

class Solution:
    def permuteUnique(self, nums: List[int]) -> List[List[int]]:

        used=[False]*len(nums)

            if used[i]:

            self.backtracking(nums,path,res,used)
